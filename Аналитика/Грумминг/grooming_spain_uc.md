# Use Cases для системы груминга в Испании

## UC-001: Бронирование процедуры груминга с дополнительными услугами

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-001. Бронирование процедуры груминга с выбором дополнительных услуг |
| **User Story** | US-001: Как владелец питомца, я хочу забронировать процедуру груминга онлайн, чтобы удобно выбрать дату и время без необходимости телефонных звонков.<br>US-002: Как владелец питомца, я хочу видеть и выбирать дополнительные услуги (игры с питомцем, дневной присмотр), чтобы получить полный спектр услуг для моего питомца. |
| **Участники** | • Основной: Владелец питомца<br>• Вторичные: Система бронирования, Календарь сотрудников, Система уведомлений |
| **Предусловия** | • Владелец зарегистрирован в системе<br>• Питомец добавлен в профиль<br>• Система бронирования доступна<br>• Расписание сотрудников актуально |
| **Ограничения** | • Бронирование доступно за 30 дней вперед<br>• Максимум 3 питомца на одну запись<br>• Время процедуры: 1-4 часа в зависимости от размера питомца |
| **Триггер** | Владелец нажимает кнопку "Записаться на груминг" |
| **Основной сценарий** | 1. Система отображает календарь с доступными датами<br>2. Владелец выбирает удобную дату и время<br>3. Система показывает доступных грумеров на выбранное время<br>4. Владелец выбирает питомца из списка<br>5. Система отображает базовые услуги и цены<br>6. Владелец выбирает дополнительные услуги (игры, дневной присмотр)<br>7. Система рассчитывает итоговую стоимость<br>8. Владелец подтверждает бронирование<br>9. Система отправляет подтверждение на email<br>10. Запись добавляется в расписание грумера |
| **Альтернативные сценарии** | **п.2**: Если нет свободных слотов → показать ближайшие доступные даты<br>**п.4**: Если питомец не добавлен → предложить добавить нового питомца<br>**п.6**: Если выбрана услуга "дневной присмотр" → запросить время возврата<br>**п.8**: Если оплата требуется заранее → перенаправить на страницу оплаты |
| **Постусловия** | • Бронирование создано и подтверждено<br>• Уведомление отправлено владельцу и грумеру<br>• Запись добавлена в расписание |

## UC-002: Процесс груминга с игровой сессией

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-002. Проведение процедуры груминга с предварительной игровой сессией |
| **User Story** | US-003: Как владелец питомца, я хочу иметь возможность заказать игровую сессию перед грумингом, чтобы мой питомец расслабился и чувствовал себя комфортно перед процедурой. |
| **Участники** | • Основной: Грумер<br>• Вторичные: Питомец, Система учета, Владелец (через уведомления) |
| **Предусловия** | • Бронирование подтверждено<br>• Питомец доставлен в салон<br>• Игровая зона доступна<br>• Грумер готов к работе |
| **Ограничения** | • Игровая сессия: 15-30 минут<br>• Максимум 2 питомца одновременно в игровой зоне<br>• Использование только сертифицированных игрушек |
| **Триггер** | Питомец прибывает в салон для запланированной процедуры |
| **Основной сценарий** | 1. Грумер встречает питомца и владельца<br>2. Проверяет бронирование и выбранные услуги<br>3. Если заказана игровая сессия → проводит в игровую зону<br>4. Проводит 15-30 минут игровой сессии<br>5. Наблюдает за поведением питомца<br>6. Переводит питомца в зону груминга<br>7. Начинает основную процедуру груминга<br>8. Отправляет фото-отчет владельцу после игровой сессии |
| **Альтернативные сценарии** | **п.4**: Если питомец агрессивен → прекратить игру, начать процедуру<br>**п.4**: Если питомец слишком возбужден → сократить время игры<br>**п.7**: Если питомец не успокаивается → использовать дополнительные методы релаксации |
| **Постусловия** | • Питомец расслаблен и готов к процедуре<br>• Фото-отчет отправлен владельцу<br>• Начата основная процедура груминга |

## UC-003: Дневной присмотр за питомцем

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-003. Организация дневного присмотра за питомцем после груминга |
| **User Story** | US-004: Как владелец питомца, я хочу оставить питомца на весь день после груминга, чтобы совместить процедуру с моими делами и обеспечить питомцу безопасный присмотр. |
| **Участники** | • Основной: Владелец питомца<br>• Вторичные: Грумер, Ассистент по присмотру, Система уведомлений |
| **Предусловия** | • Процедура груминга завершена<br>• Услуга "дневной присмотр" оплачена<br>• Владелец указал время возврата<br>• Места в зоне присмотра доступны |
| **Ограничения** | • Максимальное время присмотра: 8 часов<br>• Кормление по расписанию владельца<br>• Прогулки каждые 2-3 часа<br>• Видеонаблюдение в зоне присмотра |
| **Триггер** | Завершение процедуры груминга |
| **Основной сценарий** | 1. Грумер завершает процедуру груминга<br>2. Передает питомца ассистенту по присмотру<br>3. Ассистент размещает питомца в комфортной зоне<br>4. Обеспечивает кормление по расписанию<br>5. Организует прогулки каждые 2-3 часа<br>6. Отправляет фото-отчеты владельцу каждые 2 часа<br>7. Подготавливает питомца к возврату за 15 минут до назначенного времени<br>8. Передает питомца владельцу с отчетом о дне |
| **Альтернативные сценарии** | **п.4**: Если питомец отказывается от еды → уведомить владельца<br>**п.5**: Если плохая погода → организовать активность в помещении<br>**п.7**: Если владелец задерживается → уведомить и согласовать дополнительное время |
| **Постусловия** | • Питомец находится под присмотром<br>• Владелец получает регулярные отчеты<br>• Питомец готов к возврату в назначенное время |

## UC-004: Премиальная оплата услуг

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-004. Оплата премиальных услуг груминга |
| **User Story** | US-009: Как владелец питомца, я хочу оплачивать услуги онлайн с возможностью выбора премиальных опций, чтобы получить высококачественный сервис за соответствующую стоимость. |
| **Участники** | • Основной: Владелец питомца<br>• Вторичные: Платежная система, Система бухгалтерии, Администратор |
| **Предусловия** | • Услуги выбраны и рассчитаны<br>• Владелец авторизован в системе<br>• Платежная система доступна<br>• Баланс/карта владельца активна |
| **Ограничения** | • Минимальная сумма оплаты: 50€<br>• Максимальная сумма одной транзакции: 1000€<br>• Поддерживаемые платежные методы: карты, PayPal, банковский перевод |
| **Триггер** | Владелец подтверждает выбор услуг и переходит к оплате |
| **Основной сценарий** | 1. Система отображает детализированный счет с премиальными опциями<br>2. Владелец выбирает способ оплаты<br>3. Вводит платежные данные<br>4. Система обрабатывает платеж через платежный шлюз<br>5. При успешной оплате генерирует электронный чек<br>6. Отправляет подтверждение оплаты на email<br>7. Обновляет статус бронирования на "оплачено"<br>8. Уведомляет администратора о поступлении платежа |
| **Альтернативные сценарии** | **п.4**: Если платеж отклонен → предложить альтернативный способ оплаты<br>**п.4**: Если недостаточно средств → предложить разделить платеж<br>**п.4**: Если превышен лимит → предложить оплату на месте |
| **Постусловия** | • Услуги оплачены<br>• Электронный чек сгенерирован<br>• Бронирование подтверждено<br>• Средства зачислены на счет салона |
