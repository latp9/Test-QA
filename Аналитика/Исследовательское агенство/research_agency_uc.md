# Use Cases для исследовательского агенства

## UC-001: Регистрация нового исследователя

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-001. Регистрация нового исследователя в системе агенства |
| **User Story** | US-001: Как менеджер по найму, я хочу иметь возможность регистрировать нового исследователя в системе, чтобы пополнять базу доступных специалистов для клиентов |
| **Участники** | • Основной: Менеджер по найму<br>• Вторичные: Система регистрации, База данных исследователей, Email-сервис |
| **Предусловия** | • Менеджер авторизован в системе<br>• Доступен модуль управления исследователями<br>• Кандидат прошел предварительный отбор |
| **Ограничения** | • Email исследователя должен быть уникальным<br>• Обязательные поля: ФИО, специализация, опыт работы<br>• Максимум 5 специализаций на одного исследователя |
| **Триггер** | Менеджер нажимает кнопку "Добавить исследователя" |
| **Основной сценарий** | 1. Система отображает форму регистрации исследователя<br>2. Менеджер заполняет обязательные поля (ФИО, email, телефон)<br>3. Менеджер указывает специализации и уровень опыта<br>4. Менеджер загружает резюме и портфолио<br>5. Система валидирует введенные данные<br>6. Система проверяет уникальность email<br>7. Система создает учетную запись исследователя<br>8. Система отправляет приветственное письмо исследователю<br>9. Система присваивает статус "На проверке" |
| **Альтернативные сценарии** | **п.5**: Данные некорректны → показать ошибки валидации<br>**п.6**: Email уже существует → предложить объединение записей<br>**п.8**: Ошибка отправки email → сохранить аккаунт, показать предупреждение |
| **Постусловия** | • Исследователь зарегистрирован в системе<br>• Отправлено приветственное письмо<br>• Статус исследователя: "На проверке" |

## UC-002: Обработка запроса клиента на исследователя

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-002. Обработка запроса клиента на предоставление исследователя |
| **User Story** | US-005: Как менеджер по работе с клиентами, я хочу обрабатывать запросы клиентов на предоставление исследователей, чтобы оперативно подбирать подходящих специалистов |
| **Участники** | • Основной: Менеджер по работе с клиентами<br>• Вторичные: Клиент, Система подбора, База данных исследователей |
| **Предусловия** | • Клиент зарегистрирован в системе<br>• Менеджер авторизован<br>• Доступна база свободных исследователей |
| **Ограничения** | • Срок обработки запроса: 24 часа<br>• Минимальный срок аренды: 1 месяц<br>• Максимум 10 кандидатов в предложении |
| **Триггер** | Поступление нового запроса от клиента |
| **Основной сценарий** | 1. Система уведомляет менеджера о новом запросе<br>2. Менеджер анализирует требования клиента<br>3. Система фильтрует исследователей по критериям<br>4. Менеджер просматривает подходящих кандидатов<br>5. Менеджер формирует shortlist из 3-5 исследователей<br>6. Система генерирует коммерческое предложение<br>7. Менеджер отправляет предложение клиенту<br>8. Клиент выбирает кандидата<br>9. Система обновляет статус исследователя на "Забронирован" |
| **Альтернативные сценарии** | **п.3**: Нет подходящих кандидатов → уведомить клиента о расширении критериев<br>**п.8**: Клиент отклоняет всех кандидатов → запросить дополнительные требования<br>**п.9**: Исследователь недоступен → предложить альтернативных кандидатов |
| **Постусловия** | • Запрос обработан<br>• Клиент получил предложение<br>• Статус исследователя обновлен |

## UC-003: Назначение исследователя на проект

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-003. Назначение исследователя на клиентский проект |
| **User Story** | US-008: Как менеджер проектов, я хочу назначать исследователей на конкретные проекты, чтобы обеспечить выполнение обязательств перед клиентами |
| **Участники** | • Основной: Менеджер проектов<br>• Вторичные: Исследователь, Клиент, Система учета времени |
| **Предусловия** | • Проект создан и утвержден клиентом<br>• Исследователь доступен для работы<br>• Подписан договор с клиентом |
| **Ограничения** | • Рабочее время: 40 часов в неделю<br>• Одновременно не более 3 проектов на исследователя<br>• Минимальная длительность проекта: 2 недели |
| **Триггер** | Клиент подтверждает выбор исследователя |
| **Основной сценарий** | 1. Менеджер создает задание в системе<br>2. Система проверяет доступность исследователя<br>3. Менеджер назначает исследователя на проект<br>4. Система уведомляет исследователя о новом задании<br>5. Исследователь подтверждает принятие проекта<br>6. Система создает рабочий план проекта<br>7. Менеджер устанавливает контрольные точки<br>8. Система начинает отсчет рабочего времени |
| **Альтернативные сценарии** | **п.2**: Исследователь недоступен → выбрать другого кандидата<br>**п.5**: Исследователь отклоняет проект → найти замену<br>**п.7**: Проект требует корректировки → согласовать с клиентом |
| **Постусловия** | • Исследователь назначен на проект<br>• Создан рабочий план<br>• Начат учет рабочего времени |

## UC-004: Учет рабочего времени исследователя

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-004. Учет рабочего времени исследователя по проектам |
| **User Story** | US-011: Как бухгалтер, я хочу учитывать рабочее время исследователей, чтобы корректно начислять зарплату и выставлять счета клиентам |
| **Участники** | • Основной: Исследователь<br>• Вторичные: Система учета времени, Бухгалтер, Менеджер проектов |
| **Предусловия** | • Исследователь назначен на активный проект<br>• Доступна система учета времени<br>• Установлены нормы рабочего времени |
| **Ограничения** | • Минимальная единица учета: 15 минут<br>• Максимум 10 часов в день<br>• Обязательное подтверждение менеджером |
| **Триггер** | Исследователь начинает работу над задачей |
| **Основной сценарий** | 1. Исследователь запускает таймер в системе<br>2. Система фиксирует начало работы<br>3. Исследователь выполняет задачу<br>4. Исследователь останавливает таймер по завершении<br>5. Система записывает затраченное время<br>6. Исследователь добавляет описание выполненной работы<br>7. Система формирует daily report<br>8. Менеджер проверяет и подтверждает отчет<br>9. Система передает данные в бухгалтерию |
| **Альтернативные сценарии** | **п.1**: Ручной ввод времени → валидация системой<br>**п.8**: Менеджер отклоняет отчет → требовать пояснения<br>**п.9**: Ошибка передачи данных → повторная синхронизация |
| **Постусловия** | • Время учтено и подтверждено<br>• Данные доступны для расчета оплаты<br>• Клиенту подготовлен отчет о работе |

## UC-005: Формирование финансового отчета

| **Элемент** | **Описание** |
|-------------|--------------|
| **Название** | UC-005. Формирование месячного финансового отчета |
| **User Story** | US-012: Как финансовый директор, я хочу получать финансовые отчеты по проектам и исследователям, чтобы анализировать рентабельность и принимать управленческие решения |
| **Участники** | • Основной: Финансовый директор<br>• Вторичные: Система отчетности, База данных проектов, Бухгалтерия |
| **Предусловия** | • Завершен отчетный период (месяц)<br>• Все данные по проектам собраны<br>• Проведена сверка с бухгалтерией |
| **Ограничения** | • Срок формирования отчета: 3 рабочих дня<br>• Данные за последние 3 года доступны для сравнения<br>• Отчет должен включать анализ трендов |
| **Триггер** | Начало нового месяца (1-е число) |
| **Основной сценарий** | 1. Система автоматически запускает процесс формирования отчета<br>2. Собираются данные по всем завершенным проектам<br>3. Анализируется рабочее время исследователей<br>4. Рассчитывается доходность по каждому проекту<br>5. Формируется сводная таблица по исследователям<br>6. Создаются графики и диаграммы эффективности<br>7. Система генерирует PDF отчет<br>8. Отчет отправляется финансовому директору<br>9. Данные архивируются для исторического анализа |
| **Альтернативные сценарии** | **п.2**: Отсутствуют данные по некоторым проектам → запросить у менеджеров<br>**п.7**: Ошибка генерации PDF → создать отчет в Excel<br>**п.8**: Директор недоступен → сохранить отчет в системе |
| **Постусловия** | • Финансовый отчет сформирован<br>• Данные доступны для анализа<br>• Архив пополнен новыми данными |
